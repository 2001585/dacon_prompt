## 🚨 핵심 발견사항

### **Goldilocks Zone 존재**
- 너무 복잡하면 GPT가 실수
- 너무 단순하면 GPT가 오해  
- **적정 복잡도 (900-1000자)가 최적**

### **구조의 힘**
1. 명확한 분류 (회사명/키워드)
2. 단순한 조건 ("이나" 사용)
3. 예외사항 별도 명시
4. 출력형식 명확히 지시

### **v3.3이 잘 된 이유**
✅ 길이 최적화 (973자)
✅ 구조화된 설명

✅ 명확한 출력 지시
