# 5단계: 프롬프트 최적화

## 목표
테스트 결과를 바탕으로 시스템 프롬프트를 개선하고 최적화

## 작업 내용

### 1. 오분류 분석 및 개선
- [ ] 오분류된 케이스별 원인 분석
- [ ] 분류 기준 세분화
- [ ] 키워드 가중치 조정
- [ ] 문맥 이해 능력 향상

### 2. 프롬프트 구조 최적화
- [ ] 정보 배치 순서 조정
- [ ] 중요도에 따른 내용 재편성
- [ ] 불필요한 부분 제거
- [ ] 핵심 내용 강조

### 3. 길이 최적화
- [ ] 현재 글자 수 대비 성능 분석
- [ ] 평가 공식 기반 최적점 탐색
- [ ] 압축 가능한 부분 식별
- [ ] 효율적 표현 방식 적용

### 4. A/B 테스트
- [ ] 여러 버전의 프롬프트 작성
- [ ] 각 버전별 성능 비교
- [ ] 최적 조합 탐색
- [ ] 교차 검증

### 5. 반복 개선 사이클
- [ ] 버전별 성능 추적
- [ ] 점진적 개선 적용
- [ ] 성능 정체시 대안 전략 시도
- [ ] 최종 버전 선정

## 최적화 전략
1. **정확도 우선**: 정확도가 현저히 낮다면 길이를 늘려서라도 정확도 개선
2. **길이 최적화**: 정확도가 충분하다면 길이 단축으로 길이 점수 향상
3. **균형점 탐색**: 정확도와 길이의 최적 균형점 탐색

## 산출물
- 개선된 시스템 프롬프트 (v2.0 이상)
- 버전별 성능 비교표
- 최적화 과정 문서
