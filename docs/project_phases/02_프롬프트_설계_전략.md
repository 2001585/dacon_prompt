# 2단계: 프롬프트 설계 전략

## 목표
3000자 제한 내에서 최적의 시스템 프롬프트 설계 전략 수립

## 작업 내용

### 1. 평가 지표 분석
- [ ] 평가 공식 분석: `(0.9 × 모델 분류 정확도) + (0.1 × 시스템 프롬프트 길이 점수)`
- [ ] 길이 점수와 정확도의 트레이드오프 분석
- [ ] 최적 프롬프트 길이 범위 결정

### 2. 프롬프트 구조 설계
- [ ] 역할 정의 (Role Definition)
- [ ] 작업 설명 (Task Description)
- [ ] 분류 기준 (Classification Rules)
- [ ] 출력 형식 (Output Format)
- [ ] 예시 제공 (Few-shot Examples)

### 3. 길이 최적화 전략
- [ ] 핵심 내용 우선순위 설정
- [ ] 불필요한 표현 제거 방법
- [ ] 압축된 표현 방식 개발
- [ ] 효과적인 키워드 활용

### 4. 정확도 향상 전략
- [ ] 체인 오브 생각(Chain of Thought) 적용
- [ ] 반례 처리 방법
- [ ] 애매한 케이스 판단 기준
- [ ] 오분류 방지 장치

## 산출물
- 프롬프트 구조 템플릿
- 길이 최적화 가이드
- 핵심 요소 체크리스트
